# Runs Index

Use this folder for one markdown file per run.

The purpose is not archival fluff. These files are the training corpus for later strategy and automation skills.

## Rules

- Create one file per run.
- Append after every floor.
- Do not wait until the run ends.
- Quiet floors still get logged.
- If a floor teaches a reusable lesson, link the relevant vault note and update that note too.

## Naming

Use a stable sortable name:

- `YYYY-MM-DD-<character>-run-<short-id>.md`

Example:

- `2026-03-16-ironclad-run-001.md`

## Required Per-Floor Fields

- floor number
- room type
- relevant starting state
- action or decision
- outcome
- lesson or hypothesis
