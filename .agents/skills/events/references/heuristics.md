# Heuristics

Use this note before making event choices.

## Core Rule

Do not evaluate an event in isolation from the route, relic stack, and next threshold check.

## Event Classification

Before choosing, classify the event:

- damage-control event
- value event
- deck-quality event
- pathing enabler
- hard-fight prep
- trap disguised as value

That classification should change what counts as the best option.

## What To Rebuild From Context

Always re-check:

- HP and max HP
- gold
- deck shape and current bloat
- relic context
- potion slack
- upcoming route pressure
- the run's current bottleneck

## Practical Rules

- Generated cards must be judged against the relic stack and current engine, not by the flashiest standalone preview.
- A value event should often push the run toward safer pathing rather than justify immediate greed.
- A deterministic remove, transform, or clean gold gain is often better than vague upside.
- If the event text is not concrete, fix the exporter before choosing.

## Hidden Cost Checks

Ask what each option worsens:

- current HP safety
- deck compactness
- future hand quality
- potion flexibility
- elite or boss readiness

Short-term value that breaks the next threshold check is usually fake value.

## Failure Modes To Avoid

- taking a reward that points one way, then pathing as if a different reward had been taken
- ignoring relic context on generated-card events
- treating new STS2 events like solved STS1 events without reading the actual text

## Living Rule

Keep this note broad and evolving. Merge repeated event lessons here instead of spawning lots of tiny event meta-notes.
